# Spring Boot Backend API Rest Project for coherent Test Task of Francisco Valdez 

This project provides a RESTful API for managing reservations.

## Technologies Used

This reservation API is built using the following technologies:

1. **Spring Boot**: 2.7.11
   Spring Boot is a framework that simplifies the development of Java applications by providing pre-configured settings and conventions.

2. **MySQL DB**: 8.0.33
   This project is persisted into a MySQL DB hosted in a RDS AWS.

3. **Hibernate Core**: 5.5.7.Final
   Hibernate is an object-relational mapping (ORM) framework that provides mapping between Java objects and relational databases.

4. **Hibernate EntityManager**: 5.5.7.Final
   Hibernate EntityManager is a subproject of Hibernate that provides additional features for managing entities in a JPA (Java Persistence API) context.

5. **Hibernate Annotations**: 3.5.6-Final
   Hibernate Annotations is a library that allows developers to use annotations to define the mapping between Java classes and database tables.

6. **Spring Boot Starter Data JPA**
   Spring Boot Starter Data JPA is a starter package that provides auto-configuration for Spring Data JPA, making it easier to work with JPA repositories.

7.  **Lombok**: 1.18.20
    Lombok is a library that helps reduce boilerplate code in Java classes by providing annotations to generate getters, setters, constructors, and other common methods.

8.  **Springfox Boot Starter**: 3.0.0
    Springfox Boot Starter is a library that integrates Swagger with Spring Boot, allowing for API documentation generation.

9.  **Springfox Swagger UI**: 3.0.0
    - Description: Springfox Swagger UI is a library that provides a user interface for browsing and testing the API documentation generated by Swagger.

## API Endpoints

The following RESTful endpoints are available in the Betask Reservation API:

- `GET /api/reservation`: Retrieves all reservation records.
- `POST /api/reservation`: Creates a new reservation record.
- `PUT /api/reservation/{id}`: Updates the reservation record with the specified ID.
- `DELETE /api/reservation/{id}`: Deletes the reservation record with the specified ID.

For detailed information on how to use each endpoint, refer to the API documentation generated by Swagger.

## Getting Started

To get started with the Betask Reservation API, follow these steps:

1. Clone the repository.
2. Configure the database connection in the application properties file.
3. Build the project using Maven.
4. Run the application.
5. Access the API endpoints using a REST client or browser.
